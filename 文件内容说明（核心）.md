## 说明

本文件用于说明 RAG-Anything 的核心文件内容，帮助理解 RAG-Anything 的实现原理和使用方法，后续为后端提供接口时，可以参考这些文件的实现方式

### config.py 配置文件

- 包含 RAGAnythingConfig 类，用于配置 RAG-Anything 的各种参数
- 可以配置的参数包括：
    - 工作目录：RAG 存储和缓存文件的存放目录
    - 解析器配置
        - 文档解析方法
        - 解析后的内容的默认输出目录
        - 解析器选择(mineru, docling, paddleocr)
        - 是否在解析时显示内容统计信息
    - 多模态处理配置
        - 是否启用图像处理
        - 是否启用表格处理
        - 是否启用公式处理
    - 批量处理配置
        - 最大同时并行处理的文件数量
        - 批量处理模式下支持的文件扩展名列表
        - 是否递归处理文件夹
    - 上下文提取配置
        - 上下文窗口大小
        - 基于页面还是数据块提取上下文
        - 最大上下文 Token 数量
        - 是否包含页眉和标题
        - 是否包含图片/表格的说明文字
        - 上下文过滤内容类型
        - 默认内容格式
    - 路径处理配置：是否在 LightRAG 文件引用中使用全路径或仅使用文件名

### processor.py 文件

- 包含RAGAnything 的文档处理功能，包含用于解析文档及处理多模态内容的方法

#### `process_document_complete()`

```
process_document_complete(
            self,
            file_path: str,
            output_dir: str = None,
            parse_method: str = None,
            display_stats: bool = None,
            split_by_character: str | None = None,
            split_by_character_only: bool = False,
            doc_id: str | None = None,
            file_name: str | None = None,
            **kwargs,
    )
```

- 作用：完整文档处理流水线，包括解析、入库、多模态处理等
- 参数说明：
    - file_path: 待处理文件的路径
    - output_dir: 输出目录（默认为配置中的 config.parser_output_dir）
    - parse_method: 解析方法（默认为配置中的 config.parse_method，如 'auto', 'ocr'）
    - display_stats: 是否显示内容统计信息（默认为配置中的 config.display_content_stats）
    - split_by_character: 可选的特定分隔符，用于对提取出的文本进行切分
    - split_by_character_only: 如果设为 True，则仅使用指定的字符进行切分，忽略默认切分规则
    - doc_id: 可选的文档 ID。如果不提供，系统会根据文件内容自动生成唯一的 ID
    - file_name: 用于引用溯源的文件名（默认为 None，将从 file_path 中提取）
    - **kwargs: 解析器的额外参数。例如：lang（语言）、device（设备，如 cpu/cuda）、start_page/end_page（起始/结束页码）、
      formula（公式识别开关）、table（表格识别开关）、backend（后端引擎）、source（来源标识）

#### `parse_document()`

```
parse_document(
            self,
            file_path: str,
            output_dir: str = None,
            parse_method: str = None,
            display_stats: bool = None,
            **kwargs,
    ) -> tuple[List[Dict[str, Any]], str]
```

- 作用：带缓存支持的文档解析函数
- 参数说明：
    - file_path: 待解析文件的路径
    - output_dir: 输出目录（默认为配置中的 config.parser_output_dir）
    - parse_method: 解析方法（默认为配置中的 config.parse_method）
    - display_stats: 是否显示内容统计信息（默认为配置中的 config.display_content_stats）
    - **kwargs: 传递给解析器的额外参数（如语言、设备、起始页码、是否开启公式/表格识别等）
- 返回：
    - tuple[List[Dict[str, Any]], str]: (内容块列表, 文档唯一ID)

- 说明：内容块列表中每个元素都是一个字典，包含内容类型、文本、页码等信息，具体内容可以参照`变量结构.md`中的`content_list`

### query.py 文件

- 包含 RAG-Anything 的查询功能，包含用于文本查询和多模态查询的方法

#### `aquery()`

```
aquery(
    self, query: str, mode: str = "mix", system_prompt: str | None = None, **kwargs
) -> str
```

- 作用：纯文本查询
- 参数说明：
    - query: 查询文本
    - mode: 查询模式 ("local", "global", "hybrid", "naive", "mix", "bypass")
    - system_prompt: 可选的系统提示词
    - **kwargs: 其他查询参数，将传递给 QueryParam

#### `aquery_with_multimodal()`

```
aquery_with_multimodal(
            self,
            query: str,
            multimodal_content: List[Dict[str, Any]] = None,
            mode: str = "mix",
            **kwargs,
    ) -> str
```

- 作用：多模态查询
- 参数说明：
    - query: 基础查询文本（用户的提问）
    - multimodal_content: 多模态内容列表，每个元素包含：
        - type: 内容类型（如 "image"、"table"、"equation" 等）
        - 其他字段取决于类型（例如 img_path 图片路径、table_data 表格数据、latex 公式等）
    - mode: 查询模式 ("local", "global", "hybrid", "naive", "mix", "bypass")
    - **kwargs: 其他查询参数，将传递给 QueryParam

### RAGAnything类

使用示例

```
rag = RAGAnything(
            config=config,  # RAGAnything的配置，通过RAGAnythingConfig进行配置
            llm_model_func=llm_model_func,  # LLM模型函数，用于文本分析
            vision_model_func=vision_model_func,  # 视觉模型函数，用于图像分析
            embedding_func=embedding_func,  # 嵌入函数，用于将文本转换为数值向量
        )
```

### batch_parser.by 文件

- 说明：这个py文件包含批量并行处理引擎，是整个项目进行并行处理的核心文件

#### class BatchProcessingResult

- 说明：一个类，用于存储文件处理后产生的结果
- 包含的参数：
    - successful_files: List[str] 成功处理的文件
    - failed_files: List[str] 处理失败的文件
    - total_files: int 所有处理文件的数目
    - processing_time: float 处理所花的时间
    - errors: Dict[str, str] 产生的错误
    - output_dir: str 输出目录
    - dry_run: bool = False 是否干运行

#### class BatchParser

- 说明：核心类，通过创建实例来使用其中的方法，从而可以进行文件的并行处理
- 初始化参数：
    - parser_type: 指定要使用的解析器，可以为mineru, docling, 或者 paddleocr
    - max_workers: 指定最大的并行工作线程数
    - show_progress: 是否显示进度条
    - timeout_per_file: 每个文件的超时时间（秒）
    - skip_installation_check: 是否跳过解析器安装检查（适用于测试，实际使用时应当进行检查）

##### process_single_file()

```
process_single_file(
        self,
        file_path: str,
        output_dir: str,
        parse_method: str = "auto",
        **kwargs
    ) -> Tuple[bool, str, Optional[str]]:
```

- 说明：这个函数用于处理单个文件
- 参数说明：
    - file_path: 要处理的文件路径
    - output_dir: 输出目录
    - parse_method: 解析方法，比如auto，txt，ocr，默认为auto
        - 解析方法说明：auto自动识别文件类型，适用于所有文件；txt用于解析纯文本；ocr用于解析图片和扫描件
    - **kwargs: 其他解析器参数
- 返回：
    - 一个(成功, 文件路径, 错误信息) 的元组
    - 分别说明处理是否成功，被处理文件所在的路径（不是输出目录），以及当处理失败时的错误信息（处理成功则为None）

##### process_batch()

```
process_documents_batch(
        self,
        file_paths: List[str],
        output_dir: Optional[str] = None,
        parse_method: Optional[str] = None,
        max_workers: Optional[int] = None,
        recursive: Optional[bool] = None,
        show_progress: bool = True,
        **kwargs,
    ) -> BatchProcessingResult:
```

- 说明：用于批量并行处理文件的函数（同步多线程）
- 参数说明：
    - file_paths: 要处理的文件路径或目录的列表
    - output_dir: 基础输出目录
    - parse_method: 所有文件的解析方法，auto，txt，或ocr
    - recursive: 是否递归搜索目录
    - dry_run: 当为 True 时，仅列出文件而不处理它们
    - **kwargs: 其他解析器参数
- 返回：一个包含处理统计信息的 BatchProcessingResult 实例

##### process_batch_async()

```
process_batch_async(
        self,
        file_paths: List[str],
        output_dir: str,
        parse_method: str = "auto",
        recursive: bool = True,
        dry_run: bool = False,
        **kwargs,
    ) -> BatchProcessingResult:
```

- 说明：process_batch()的异步版本，同样可以批量并行的处理文件，
        区别在于系统可以在这个函数运行时去处理别的任务，无需等待这个函数运行结束
        这个函数的实现本质是调用process_batch()，只是将其放入线性池当中实现异步运行
- 参数说明：(跟process_batch的参数一样)
    - file_paths: 要处理的文件路径或目录的列表
    - output_dir: 基础输出目录
    - parse_method: 所有文件的解析方法
    - recursive: 是否递归搜索目录
    - dry_run: 当为 True 时，仅列出文件而不处理它们
    - **kwargs: 其他解析器参数
- 返回：一个包含处理统计信息的 BatchProcessingResult 实例
